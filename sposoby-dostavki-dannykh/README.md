# Способы доставки данных

## Способы доставки данных для инженера данных

Инженеры данных используют два основных подхода для доставки данных из источников в централизованное хранилище для анализа: ETL (Extract-Transform-Load) и ELT (Extract-Load-Transform). Оба метода состоят из трех базовых этапов, но различаются порядком выполнения и тем, как они реализуются.

### ETL (Extract-Transform-Load)

1. **Извлечение (Extract)**: Данные копируются из различных источников, таких как базы данных SQL/NoSQL, файлы, облачные приложения и т.д. Извлечение может быть полным или инкрементальным.

2. **Преобразование (Transform)**: Извлеченные данные очищаются, валидируются, структурируются и преобразуются в соответствии с требованиями целевой системы. Преобразования выполняются во временном хранилище перед загрузкой в финальное.

3. **Загрузка (Load)**: Преобразованные данные загружаются в целевое хранилище данных. Это может быть физическая вставка записей или пакетная загрузк.

### ELT (Extract-Load-Transform) 

1. **Извлечение (Extract)**: Данные копируются из источников в целевую систему хранения, такую как хранилище данных или озеро данных.

2. **Загрузка (Load)**: Извлеченные "сырые" данные напрямую загружаются в целевое хранилище без преобразований.

3. **Преобразование (Transform)**: Уже загруженные данные в целевой системе очищаются, валидируются, структурируются и преобразуются по мере необходимости. Преобразования могут выполняться с помощью SQL или фреймворков обработки данных.

## Сравнение ETL и ELT

**Преимущества ELT**:
- Гибкость: данные сначала загружаются "сырыми", а преобразования выполняются позже по мере необходимости
- Масштабируемость: современные хранилища данных и озера данных позволяют хранить неограниченные объемы данных
- Вовлечение аналитиков: аналитики могут помогать инженерам в преобразованиях, используя SQL

**Преимущества ETL**:
- Контроль качества: преобразования выполняются до загрузки, что позволяет контролировать качество данных
- Производительность: преобразованные данные загружаются в оптимизированном для аналитики виде

Выбор между ETL и ELT зависит от конкретных требований проекта, объема данных, используемых технологий и предпочтений команды. Современные инструменты поддерживают как ETL, так и ELT подходы, позволяя инженерам данных гибко выбирать наиболее подходящий вариант.
