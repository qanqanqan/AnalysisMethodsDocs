### Компактизация в HBase:

**Компактизация** — это процесс объединения нескольких файлов **HFile** для улучшения производительности и освобождения места. Существует два типа:

1. **Малая компактизация (Minor compaction)**:
   - Объединяет несколько небольших HFiles в один файл, чтобы уменьшить их количество.
   - Не удаляет устаревшие версии данных или пометки на удаление ("tombstones").

2. **Большая компактизация (Major compaction)**:
   - Сливает все HFiles региона в один.
   - Удаляет устаревшие версии данных и "tombstones", освобождая место.

Компактизация улучшает производительность чтения, снижая количество файлов, которые нужно просматривать для поиска данных.

### Репликация данных в HBase:

**Репликация** — это механизм копирования данных между кластерами для обеспечения отказоустойчивости, высокой доступности и балансировки нагрузки.

1. **Асинхронная репликация**:
   - Данные записываются в журнал WAL в основном кластере, а затем асинхронно передаются в другой кластер.
   - Подходит для географически распределённых систем и балансировки чтений между кластерами.

2. **Использование репликации**:
   - **Отказоустойчивость**: В случае сбоя основного кластера можно переключиться на резервный.
   - **Балансировка нагрузки**: Чтение данных можно распределить по нескольким кластерам.
   - **Анализ данных**: Вторичный кластер может использоваться для аналитических задач без нагрузки на основной.
